Live-Sound Safety-Net · Midas M32C / X32 family

Acoustic Trio AI Mix-Assistant

A deliberately small system that listens to the room and nudges the console — doing the handful of things an absent sound engineer gets wrong, reliably and reversibly. The console does all the real audio; the app only observes and corrects within hard limits.

Python program · ~handful of files ✕ No LLM / no machine learning ✕ No MCP needed ✓ OSC + JACK/ASIO only ✓ Human override always in hand

The core principle — listen and control are separate paths

zero added latency to your audio
Console = source of truth
Analog in → mix bus → FOH
All audio & parameter state live on the M32C. The app never passes audio and never holds authoritative state.
Listen path (analysis only)
Audio in via USB / card
8 channel taps + a dedicated FOH measurement mic. Read-only — it can never sit in your signal chain.
Control path
OSC out over Ethernet (UDP 10023)
Small, bounded nudges back to faders / gains / EQ. A human on the iPad runs in parallel and wins instantly.

Pre-show — pink-noise room calibration

tunes the system to the room before doors
1 · Emit
Play ~10 s pink noise
A known reference out the main bus.
2 · Capture
Record at the meas mic
= the real room + PA spectrum in the air.
3 · Baseline
Compare to ideal pink
The room-response curve (its “normal”).
4 · Watch-list
Rank resonant peaks
The exact feedback-prone frequencies.
5 · Auto-apply
Cut peaks + pre-dip
Flatter room, fewer rings before anyone sings.
Why it's the “smart” part: the baseline stops a room's normal 250 Hz boom from being mis-flagged as feedback, while the watch-list pre-loads where feedback will start — so the live detector reacts a beat sooner there. That's the system's “learned-this-room” prior, standing in for any machine learning.

The four live jobs — two tiers by reaction time

reflex < ~85 ms · advisory ~0.5 s
🔁

Feedback catch-and-notch

Reflex

Detect a narrowband ring that is rising and frequency-stable (a held vocal note wanders and doesn't climb — so it's ignored). Park a sharp EQ cut on it. Watch-list frequencies fire one block sooner.

# confirm over consecutive blocks, then:
set_eq_notch(ch, band, freq, −9 dB, Q 8)
🛡️

Clip / overload protection

Reflex

When a channel peaks within 1 dB of full scale, trim its preamp gain down a touch so nothing distorts. Pure threshold reflex — no judgement required.

# peak > −1 dBFS →
nudge_gain_db(ch, −2 dB)
🎤

Lead-vocal level riding

Advisory

Hold the lead vocal at a loudness target as input drifts. A clamped proportional controller: error = target − measured, with a deadband, a per-step limit, and a smooth ramp so it never jumps.

# |target − avg| > 2 dB →
ramp_fader_db(lead, step ≤ ±1 dB, 350 ms)
⚖️

Balance hold

Advisory

Recall a good starting scene and gently pull the mix back when it wanders. Same clamped-controller pattern, regulating several channels toward a stored snapshot.

# spec'd; not yet in the reference code —
recall_scene() + slow drift correction

Guardrails — the layer every move passes through

the AI may never move outside these

Bounded · smoothed · logged · reversible

The policies only propose; this layer disposes. No rule can exceed a hard fader range, oversize a single step, or move audibly fast — and every action is timestamped and undoable.
FADER_MIN/MAX = −12 … +6 dB MAX_STEP = 1 dB / move RAMP = 350 ms (no jumps) notch ≤ −9 dB, narrow Q every move → decision log human override wins instantly

Build & rollout order

Step 0 · Mic + calibration
Patch the meas mic to Ch 8; run pink-noise read-only, then enable auto-correction.
Step 1 · Prove listening (no control)
Feature extractor + per-channel level & feedback lights. Push nothing yet.
Step 2 · Add OSC control (bench)
Verify exact scalings with a sniffer; test ramps & a notch with the PA off.
Step 3 · Gig as safety-net only
Auto-engage feedback + clip. Vocal-ride / balance stay advisory. iPad in hand.
Step 4 · Enable ride & balance
Set targets at soundcheck; turn on each once you trust it.
